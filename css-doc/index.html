# 重绘与回流（重排）

文章

  • 重绘：一个元素外观的改变所触发的浏览器行为，浏览器会根据元素的新属性重新绘制，使元素呈现新的外观。
    • 触发重绘条件：改变元素外观属性：color，background-color，box-shadow 等
  • 回流（重排）：当渲染树中的一部分 (或全部) 因为元素的规模尺寸，布局，隐藏等改变而需要重新构建，这就称为回流 (reflow)。每个页面至少需要一次回流，就是在页面第一次加载的时候
    • 触发重排条件：width，display，overflow
  • 重绘和回流的关系：在回流的时候，浏览器会使渲染树中受到影响的部分失效，并重新构造这部分渲染树，完成回流后，浏览器会重新绘制受影响的部分到屏幕中，该过程称为重绘。所以，重排必定会引发重绘，但重绘不一定会引发重排。

# 什么是 BFC

  • BFC 简称 块级格式化上下文 ，布局是一个完全独立的空间，让空间的子元素不会影响到外面的布局

# 简单说下浏览器解析 HTML 的整个过程

  • 浏览器通过 HTML parse 根据深度遍历的方式把 html 节点遍历成 DOM 树
  • 将 css 解析成 CSSOM 树
  • 将 DOM 树和 CSSOM 树结合，生成一个渲染树
  • 根据渲染树计算绘制每个节点的局部信息，得到布局树
  • 根据布局树绘制每个节点的像素信息，得到最终的页面图像

# flex 布局和 grid 布局区别

  • flex 布局属于一维布局，只能处理行或列适合做局部布局，例如导航栏组件
  • grid 布局属于二维布局，可以同时使用行和列，适合做整个页面的布局，比如网格状的界面
  • flex 布局是基于内容的，子元素的大小和位置由他们的内容决定
  • grid 布局是基于布局的，子元素的大小和位置由网格线和网格区域决定的。
