# JS 中 a++, ++a 如何理解？

  • a++: 执行表达式后在进行自增运算，返回的 a 为 a+1

  • ++a: 进行自增运算后在执行表达式，返回的 a 为 a+1

  • 栗子：

    • 计算 a+++a 时，按照计算从左至右的顺序，先 push (a++), 相当于 push (1),a 自增成 2 可以想象成 push (a++)，a++ 就相当于一个独立的表达式，它执行完这个表达式后进行自增运算，因此在编译器里 a 已经进行自增得到 2，接着 push (a), 相当于 push (2), 最后 pop () 出两个操作数，得到 pop ()+pop ()，即是 1+2=3;
    • 计算 a+a++ 时，按照计算从左至右的顺序，先 push (a), 相当于 push (1)，接着 push (a++), 相当于 push (1) 之后 a 自增为 2，最后 pop () 出两个操作数，得到 pop ()+pop ()，即是 1+1=2;

# 知道有几种存储方式？都有什么区别？

  • cookie, localStorage, sessionStorage, indexdDB 等
名称存储大小数据有效期作用域通信应用场景
cookie一般不超过 4k一般由服务器生成，可以设置失效时间，若没有设置时间，关闭浏览器 cookie 失效，如果设置了时间，cookie 就会存储在硬盘中，过期失效同源窗口中共享在浏览器和服务器之间来回传递，cookie 保存过多易造成性能问题判断用户是否登录过网站，以便下次自动登录或记住密码
sessionStorage5M 甚至更多仅在当前浏览器窗口关闭之前有效同一个浏览器窗口是共享的（不同浏览器，即使统一页面也不共享）仅在客户端中保存，不参与服务器的通信敏感账号一次性登录，单页面用的较多
localStorage5M 甚至更多永久有效，窗口或者浏览器关闭也会一直存在，除非手动删除在所有同源窗口中共享仅在客户端中保存，不参与服务器的通信用于长期登录，适于长期保存在本地的数据
indexdDB根据自己硬盘大小永久有效，窗口或者浏览器关闭也会一直存在，除非手动删除在所有同源窗口中共享仅在客户端中保存，不参与服务器的通信用于存储长期保存在本地较大的数据

# 前端设置 cookie 在同样地址不同的浏览器中 cookie 会同步吗？

  • 不可以

# 设置 sessionStorage 在当前页面打开一个同源的窗口，sessionStorage 共享吗？

  • 不共享，但打开的新页面可以拿到一次当前 sessionStorage 的数据，之后互不干涉。

# 一个页面中嵌入 iframe，2 个页面的 sessionStorage 数据共享吗？

  • 共享，属于同一个窗口

# cookie 可以监听改变吗？

# 构造器的 return

  • 通常构造器是没有 return 的，他们的任务是将所有必要的东西写入 this，并自动转换为结果。
  • 如果带有 return：
    • return 返回的是一个对象，则返回这个对象，而不是 this
    • return 返回的是一个原始类型，则忽略

# 浅拷贝和深拷贝有什么区别

  • 浅拷贝是拷贝一层，属性为对象时，浅拷贝是复制，两个对象指向同一个地址
  • 深拷贝是递归拷贝深层次，属性是为对象时，会新开辟一个栈，两个对象指向不同的地址

# 什么是闭包，在哪些场景中使用

  • 闭包让你可以在一个内层函数中访问到其外层函数的作用域
  • 场景：
    • 创建私有变量
    • 延长变量的声明周期

      一般函数的词法环境在函数返回后就被销毁，但是闭包会保存对创建时所在词法环境的引用，即便创建时所在的执行上下文被销毁，但创建时所在词法环境依然存在，以达到延长变量的生命周期的目的

# JS 垃圾回收机制

  • js 具有自动垃圾回收机制，垃圾收集器会定期找出那些不在使用的变量，然后释放其内存。
    • 标记清除
      • 当一个变量进入环境的时候，被标记 ' 进入环境 '，在此期间占用内存，离开环境的时候标记为 ' 离开环境 '，等到垃圾收集器运行的时候会释放带有 ' 离开环境 ' 的变量。
    • 引用计数
      • 当声明了一个变量，并用一个引用类型的值赋值给变量，则这个值的引用次数为 1, 如果包含了对这个值引用的变量又取得了另外一个值，则原先的引用值引用次数就减 1，当垃圾收集器运行的时候会将标记次数为 0 的值进行内存释放。

# == , === 和 Object.is () 分别有什么区别

  • == 会转换类型比较，宽松模式
    • '' == 0 // true
    • '0' == 0 // true
    • null == undefined // true
  • === 使用类型比较，严格模式
    • NaN === NaN // false
    • +0 === -0 // true
  • Object.is 相对比 == === 的改进
    • Object.is(NaN, NaN) // true
    • Object.is(-0, +0) // false

# new 操作符做了什么

  • 创建一个新的对象 obj

  • 将对象与构建函数通过原型链连接起来

  • 将构建函数中的 this 绑定到新建的对象 obj 上

  • 根据构建函数返回类型作判断，如果是原始值则被忽略，如果是返回对象，需要正常处理

  • 手写 new

# bind、call、apply 区别

  • apply 接收两个参数，第一个参数是 this，第二个参数接收一个数组参数，改变 this 后会立即执行一次

  • call 接收两个参数，第一个参数是 this，第二个参数接收一个参数列表，改变 this 后会立即执行一次

  • bind 接收两个参数，第一个参数是 this，第二个参数接收一个参数列表，改变 this 后不会立即执行，返回一个永久改变 this 指向函数

  • 手写一个 bind
